# コアシステム仕様書

WorldTrigger Mod — Core System Specification v1.1

---

## 1. トリオンシステム

### 概要

通常のMinecraftの体力（ハートゲージ）を廃止し、独自の**トリオンゲージ**に置き換える。
トリオンゲージは体力と技コストを兼ねた単一のリソース。

### トリオン量の計算

```
実効トリオン総量 = 基礎トリオン量 − Σ(装備中の全武器の装備コスト)
攻撃力倍率      = 実効トリオン総量 × attack_power_multiplier
```

- 基礎トリオン量はconfig.ymlで設定（デフォルト: 100）
- 武器を装備しているだけでトリオン総量が減る
- 武器が少ない = トリオン総量が多い = 攻撃力が上がる

### トリオンの増減条件

| 条件 | 変化 |
|---|---|
| 攻撃を受ける | 減少（ダメージ量に応じて） |
| 武器を発動する | 減少（武器ごとの消費トリオン） |
| バッグワーム装備中 | 毎秒減少（継続消費） |
| 戦闘中 | 回復しない |

### ベイルアウト

トリオンがゼロになった場合：
- プレイヤーはその場で消滅（ベイルアウト演出）
- スポーン地点またはベースに戻される
- バニラのデスと同じ扱いだがアイテムはドロップしない

## 2. キャラクターステータス

### 基本ステータス（デフォルト値）

| パラメータ | 値 | 実装方法 |
|---|---|---|
| 移動速度 | バニラの1.5倍 | Speed効果を常時付与 |
| ジャンプ力 | バニラの2.0倍 | Jump Boost効果を常時付与 |
| 基礎トリオン量 | 100 | カスタムデータとして管理 |

### ステータスの適用タイミング

- プレイヤーがスポーン・リスポーンしたとき
- configが再読み込みされたとき
- 毎tick確認して剥がれていたら再付与（保険）

## 3. 操作システム

### 入力マッピング

| 操作 | 動作 |
|---|---|
| 左クリック | **オフハンド**（盾スロット）の武器を通常発動 |
| 右クリック | **ホットバー選択中スロット**（メインハンド）の武器を通常発動 |
| シフト＋左クリック | オフハンドの武器の特殊技発動 |
| シフト＋右クリック | メインハンドの武器の特殊技発動 |
| スクロール / 数字キー | ホットバーのスロット切り替え（バニラ）→ 右手武器の切り替えに使う |
| Qキー（C2Sパケット） | オプショントリガースロットの武器を発動 |

### 入力検知の実装方針

- `AttackEntityCallback` / `MixinHandSwing` で左クリックを検知 → `player.getOffHandStack()` の武器を発動
- `UseBlockCallback` / `UseEntityCallback` / `UseItemCallback` で右クリックを検知 → `player.getMainHandStack()` の武器を発動
- 右クリックの三重発火を防ぐため `rightClickHandledThisTick` フラグで管理
- シフト状態は `player.isSneaking()` で判定
- Qキーはクライアント側でGLFWポーリング→C2Sパケット送信→サーバー側で発動
- 武器アイテムは `WeaponType` enumで種別を管理（NBTタグ不使用）

### クールダウン管理

- 武器ごとにクールダウンをtick単位で管理
- プレイヤーのカスタムデータに `Map<WeaponType, Integer>` で残りtickを保存
- 毎tick残りtickをデクリメント
- クールダウン中は発動不可（エラー音を鳴らす）

## 4. ロードアウトシステム

### スロット構成

```
[ホットバー（0〜8）] ← 右手武器: 数字キー/スクロールで切り替え
[オフハンドスロット] ← 左手武器: 常時1枠
[オプショントリガースロット（×6）] ← 専用インベントリ。Qキーで発動
```

- ホットバーで選択しているスロットのアイテムが右手武器になる
- オフハンドスロット（バニラの盾スロット）が左手武器になる
- オプショントリガースロットは専用インベントリ（`OptionTriggerInventory`）で管理。`PlayerScreenHandler` のMixinでスロット46〜51として追加。シフトクリックでインベントリとの移動が可能
- オプショントリガースロットにはオプション系武器（`WeaponType.isOption() == true`）のみ配置可能

### 装備制限

| 制限 | 内容 |
|---|---|
| 右手（ホットバー） | 片手・両手どちらでも装備可 |
| 左手（オフハンド） | 片手武器のみ |
| 両手武器ルール | ホットバーで両手武器を選択中は、オフハンドの武器が自動的にインベントリへ退避（満杯時はドロップ） |
| オプション系 | 武器スロットとは別枠（Qキーで使用）。最大6個同時装備 |

### 装備変更時の処理

1. 装備コストを再計算
2. 実効トリオン総量を更新
3. 攻撃力倍率を更新

## 5. クリティカルシステム

### 基本仕様

ヘッドショット判定の代わりにクリティカル率で再現する。

```
最終ダメージ = 基礎ダメージ × 攻撃力倍率 × クリティカル倍率（発生時）
```

### クリティカル率の変動条件

| 条件 | 変化量 |
|---|---|
| 基礎クリティカル率 | +10%（config.ymlで変更可） |
| スナイパー系武器使用 | +20% |
| 0.5秒以上静止して狙う | +15% |
| 移動しながら射撃 | −10% |
| バイパー使用時 | −5% |

### クリティカルダメージ倍率

- デフォルト: 2.0倍（config.ymlで変更可）
- トリオン量が多いプレイヤーほど倍率が上がる仕様は未実装（将来対応）

## 6. HUD表示

### トリオンゲージ

- 画面左下にトリオンゲージを表示
- バニラのハートゲージを非表示にする
- ゲージの色: トリオン量に応じて変化（多い=青、少ない=赤）
- 数値表示: 現在値/最大値

### クールダウン表示

- 各武器スロットのアイコン上にクールダウンのオーバーレイを表示
- バニラのアイテムクールダウン表示を流用できるか検討

---

## 7. 実装の注意事項

- バニラのダメージシステムを無効化し、トリオンダメージに置き換える
- `DamageSource` を独自のトリオンダメージソースとして定義する
- ネザーやエンドなど、特殊バイオームでの挙動は未定義（通常ワールドのみ対象）
- PvEは想定外。トリガーシステムはPvP専用として実装する
